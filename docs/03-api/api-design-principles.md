# API Design Principles

## الهدف
توحيد تصميم واجهات برمجة التطبيقات (API) في EduCore لضمان الاتساق، وقابلية التوسع، وسهولة التكامل مع تطبيقات الويب والجوال والأنظمة الخارجية.

## المبادئ الأساسية
- REST-first architecture.
- JSON هو تنسيق البيانات القياسي.
- استخدام HTTPS فقط.
- جميع الموارد تستخدم UUID كمُعرّف.
- تصميم Stateless.

## قواعد التسمية
- استخدام أسماء الموارد بصيغة الجمع.
- أمثلة:
  - GET /api/v1/students
  - POST /api/v1/students
  - GET /api/v1/students/{id}

## الاستجابات
- استخدام أكواد HTTP القياسية.
- تغليف الاستجابة في هيكل موحد.

## الأخطاء
- إرجاع Error Code وMessage وTraceId.
- عدم كشف تفاصيل داخلية للنظام.

## الإصدارات
- استخدام Versioning عبر المسار: /api/v1/
- الحفاظ على التوافق مع الإصدارات السابقة قدر الإمكان.
