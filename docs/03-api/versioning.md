# API Versioning

## الهدف
تحديد سياسة واضحة لإدارة إصدارات واجهات برمجة التطبيقات في EduCore لضمان التوافق مع العملاء الحاليين وإتاحة تطوير الميزات الجديدة دون كسر التكاملات القائمة.

## المبادئ
- استخدام الإصدار ضمن المسار: `/api/v1/`.
- الحفاظ على التوافق العكسي قدر الإمكان.
- التغييرات الكاسرة (Breaking Changes) تتطلب إصدارًا رئيسيًا جديدًا.
- إصلاحات الأخطاء والتحسينات غير الكاسرة تتم داخل نفس الإصدار.

## قواعد العمل
- VER-0101: كل Endpoint يجب أن ينتمي إلى إصدار محدد.
- VER-0102: يجب توثيق جميع التغييرات بين الإصدارات.
- VER-0103: يتم الإعلان عن إيقاف الإصدارات القديمة قبل إزالتها بمدة مناسبة.
- VER-0104: يجب اختبار جميع الإصدارات المدعومة ضمن خطط الاختبار.

## أمثلة
- GET /api/v1/students
- POST /api/v1/enrollments
- GET /api/v2/students (عند وجود تغييرات كاسرة مستقبلاً)
