# EduCore — Design Brief for Claude Design

## عن المشروع
EduCore

منصة
ERP

عربية لإدارة المراكز التعليمية
(مدارس ومعاهد)

في المملكة العربية السعودية. المستخدمون: إدارة المركز، مدرسين، موظفي استقبال ومحاسبين، وأولياء أمور. المنصة تدعم عدة فروع وعدة أعوام دراسية.

## المبادئ التصميمية الحاكمة (غير قابلة للتفاوض)
1. **العربية أولاً**
(Arabic First) — الواجهة الافتراضية عربية، الإنجليزية لغة ثانوية.
2. **RTL Native** — كل الشاشات مصممة أصلاً من اليمين لليسار، مش
LTR

مقلوب.
3. **Mobile First** — طبقاً لـ
Design System

الموثق.
4. **دعم كامل لـ WCAG 2.2 AA** — تباين ألوان مناسب، تنقل كامل بلوحة المفاتيح، توافق مع قارئات الشاشة.
5. **الوضعين الفاتح والداكن** إلزاميين لكل مكون.

## الشخصيات المستهدفة (Personas) وأولوياتهم
- **إدارة المركز / المحاسب**: شاشات كثيفة بيانات
(Dense Data)، جداول، فلترة، تقارير.
- **المدرس**: شاشات سريعة الاستخدام على الموبايل
(تسجيل حضور، رصد درجات) — أولوية لسرعة الإدخال.
- **ولي الأمر**: شاشة متابعة بسيطة جداً
(حالة الطالب، الرسوم المستحقة، الإشعارات) — أولوية للوضوح مش الكثافة.

## الوحدات المطلوب تصميمها (بالأولوية حسب خارطة الطريق)
1. **Identity & Administration** — تسجيل الدخول، إدارة الأدوار والصلاحيات.
2. **Student Management** — قائمة الطلاب، صفحة تفاصيل الطالب، نموذج إنشاء/تعديل.
3. **Academic Management** — الأعوام الدراسية، المراحل، الصفوف، الفصول.
4. **Attendance** — تسجيل حضور سريع (واجهة موبايل للمدرس تحديداً).
5. **Timetable** — جدول أسبوعي بصري.
6. **Exams & Grading** — إدخال درجات، عرض نتائج.
7. **Finance** — الفواتير، المدفوعات، كشف حساب الطالب (يشمل حالة اختيارية لعرض رمز
QR

لو الفرع مفعّل عليه
ZATCA).
8. **Reports & Dashboards** — لوحات معلومات مختلفة حسب الدور.

## مكونات Design System الأساسية المطلوبة
Buttons, Inputs, Cards, Tables, Dialogs, Navigation, Forms, Badges, Alerts, Toasts —
كل واحد لازم يدعم
RTL

والوضعين الفاتح/الداكن من أول نسخة.

## حالات لازم تتصمم لكل شاشة بيانات
- Empty State (بدون بيانات، مع رسالة إيجابية وإجراء مقترح).
- Error State (رسالة واضحة، بدون تفاصيل تقنية).
- Loading State.

## ملاحظات سياقية مهمة
- بعض التواريخ تحتاج عرض مزدوج
(ميلادي / هجري)

حسب إعداد
academic_years.calendar_display_mode.
- بعض الفروع بس هيظهر عندها عناصر
ZATCA

(QR Code على الفاتورة)، والباقي فواتير عادية — التصميم لازم يستوعب الحالتين من غير ما يبان ناقص للفروع اللي مش مفعّلة.
- لا يوجد حتى الآن نظام ألوان أو Typography نهائي معتمد — دي نقطة البداية المطلوبة من Claude Design نفسه، وليست موجودة مسبقاً.

## أول طلب مقترح لـ Claude Design
"ابدأ بتصميم نظام الألوان والـ Typography الأساسي لمنصة EduCore (عربي RTL، مركز تعليمي سعودي، وضعين فاتح وداكن)، ثم صمم شاشة تسجيل الدخول وشاشة قائمة الطلاب كنقطة انطلاق."
