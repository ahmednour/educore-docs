# Privacy & Data Protection

## الهدف
توثيق مبادئ الخصوصية وممارسات حماية البيانات المعتمدة في EduCore عبر دورة حياة المعلومات، بما يتوافق مع نظام حماية البيانات الشخصية السعودي
(PDPL).

## اعتبار خاص: بيانات القُصّر
غالبية الأشخاص أصحاب البيانات في النظام
(الطلاب)

قُصّر. طبقاً لـ
PDPL،

هذا يستوجب:
- الحصول على موافقة ولي الأمر
(Parent)

بدل الطالب نفسه على معالجة البيانات.
- تقييد الوصول لبيانات الطالب الحساسة
(الصورة، رقم الهوية)

بأدوار محددة فقط طبقاً لـ

03-api/authorization.md.
- حظر أي مشاركة لبيانات الطلاب مع أطراف خارجية دون موافقة صريحة موثقة.

## الأهداف
- حماية المعلومات الشخصية.
- تقليل جمع البيانات غير الضروري.
- تحديد قواعد الاحتفاظ والإتلاف.
- دعم متطلبات الخصوصية المطبقة.

## النطاق
- جمع البيانات.
- تخزين البيانات.
- مشاركة البيانات.
- الاحتفاظ بالبيانات.
- إتلاف البيانات.

## المبادئ
- تقليل البيانات (Data Minimization).
- تحديد الغرض (Purpose Limitation).
- الدقة.
- السرية.
- المساءلة (Accountability).

## معايير النجاح
- توثيق كيفية التعامل مع البيانات الشخصية.
- تحديد قواعد الاحتفاظ.
- مراجعة متطلبات الخصوصية بشكل دوري.
