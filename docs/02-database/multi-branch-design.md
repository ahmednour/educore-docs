# Multi-Branch Design

## الهدف
دعم تشغيل عدة فروع داخل نفس قاعدة البيانات مع ضمان عزل البيانات منطقياً وسهولة استخراج التقارير الموحدة أو الخاصة بكل فرع.

## المبادئ
- جميع البيانات التشغيلية ترتبط بـ BranchId.
- يمكن للمستخدم الوصول إلى فرع واحد أو أكثر حسب صلاحياته.
- يمكن للإدارة العامة استخراج تقارير مجمعة لجميع الفروع.

## قواعد العمل
- MB-0101: يجب أن يحتوي كل سجل تشغيلي على BranchId.
- MB-0102: يتم تطبيق التصفية حسب الفرع تلقائياً في جميع الاستعلامات.
- MB-0103: يمنع نقل البيانات بين الفروع إلا من خلال إجراءات معتمدة.
- MB-0104: يدعم النظام إضافة فروع جديدة دون الحاجة إلى تعديل بنية قاعدة البيانات.

## اعتبارات الأداء
- إنشاء فهارس على BranchId.
- استخدام Composite Index مع AcademicYearId عند الحاجة.

## التكامل
يتكامل تصميم الفروع مع نظام الصلاحيات، وسجل التدقيق، والتقارير، وجميع الوحدات التشغيلية.
