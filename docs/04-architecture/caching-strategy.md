# Caching Strategy

## الهدف
تحديد استراتيجية موحدة للتخزين المؤقت (Caching) في EduCore لتحسين الأداء وتقليل الضغط على قاعدة البيانات مع الحفاظ على اتساق البيانات.

## مستويات التخزين المؤقت
- In-Memory Cache للبيانات المحلية.
- Distributed Cache (Redis) للبيئات متعددة الخوادم.
- HTTP Response Caching عند الحاجة.

## البيانات المناسبة للتخزين المؤقت
- الإعدادات العامة.
- الفروع والمراحل والصفوف والمواد.
- الصلاحيات والأدوار.
- البيانات المرجعية قليلة التغيير.

## إبطال الكاش
- الإبطال عند تحديث البيانات.
- تحديد مدة صلاحية (TTL) مناسبة لكل نوع بيانات.
- دعم الإبطال عبر الأحداث (Event-based Invalidation) عند استخدام Redis.

## قواعد العمل
- CACHE-0101: يمنع تخزين البيانات الحساسة داخل الكاش.
- CACHE-0102: يجب أن يكون النظام قادرًا على العمل عند تعطل خدمة الكاش.
- CACHE-0103: يتم استخدام Distributed Cache في بيئات الإنتاج متعددة العقد.
- CACHE-0104: يجب مراقبة معدلات Cache Hit وCache Miss لتحسين الأداء.
