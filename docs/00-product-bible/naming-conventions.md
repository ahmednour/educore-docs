# قواعد التسمية

## الهدف
توحيد أسلوب التسمية في جميع أجزاء المشروع لضمان سهولة الصيانة والتوسع.

## الكيانات
- Student
- Parent
- Teacher
- Subject
- AcademicYear
- Stage
- Grade
- Classroom
- Enrollment
- Payment

## أسماء ممنوعة (Deprecated Aliases)
لتفادي تكرار المسميات المتضاربة بين الوثائق، الأسماء التالية ممنوع استخدامها ويُستبدل بها المقابل الرسمي:

| ممنوع | البديل الرسمي | السبب |
|---|---|---|
| Guardian | Parent | التوحيد مع 01-business وقاموس المصطلحات |
| Class | Classroom | تفادي الالتباس مع Grade |
| Fee (كمرجع للفاتورة) | Invoice | Fee يمثل نوع الرسم فقط، وليس المستند المالي |

يجب على أي مراجعة أو Pull Request رفض أي استخدام لهذه الأسماء الممنوعة.

## الملفات
- تستخدم أسماء الملفات بحروف صغيرة مع الشرطة (-).
- مثال: `academic-year.md`

## Business Rules
تبدأ جميع قواعد العمل بالبادئة `BR-` متبوعة برقم متسلسل.

مثال:
- BR-0001
- BR-0002

## Product Principles
تبدأ جميع مبادئ المنتج بالبادئة `PP-`.

## Architecture Decisions
تبدأ جميع قرارات التصميم بالبادئة `ADR-`.

## الرسائل البرمجية
تكون باللغة الإنجليزية، بينما رسائل المستخدم داخل النظام تكون باللغة العربية بشكل افتراضي.
