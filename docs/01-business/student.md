# الطالب (Student)

## الهدف
يمثل الطالب الكيان الأساسي في النظام، وترتبط به جميع العمليات الأكاديمية والمالية والإدارية. يحتفظ النظام بسجل تاريخي كامل للطالب عبر الأعوام الدراسية المختلفة.

## البيانات الأساسية
- الرقم المرجعي للطالب.
- الاسم بالعربية.
- الاسم بالإنجليزية.
- تاريخ الميلاد.
- الجنس.
- الجنسية.
- رقم الهوية أو الإقامة.
- الصورة الشخصية.
- الحالة (نشط / موقوف / منسحب / متخرج).

## العلاقات
- ولي أمر واحد أو أكثر.
- عام دراسي واحد لكل تسجيل.
- برنامج دراسي.
- مرحلة.
- صف.
- فصل.

## قواعد العمل
- BR-0801: لكل طالب رقم مرجعي فريد.
- BR-0802: لا يجوز وجود تسجيلين نشطين للطالب في نفس العام الدراسي.
- BR-0803: يجب ربط الطالب بولي أمر واحد على الأقل.
- BR-0804: يمنع حذف الطالب بعد وجود بيانات تشغيلية، ويستخدم الحذف المنطقي أو التعطيل.
- BR-0805: يسمح بنقل الطالب بين الفصول مع الاحتفاظ بالسجل التاريخي.

## معايير القبول
- إنشاء وتعديل بيانات الطالب.
- ربط الطالب بولي أمر.
- تسجيل الطالب في عام دراسي وفصل.
- الاحتفاظ بجميع السجلات التاريخية.
